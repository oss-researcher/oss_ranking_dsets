# README.md

## Overview

This document describes the **sample output files** generated by the d-set ranking pipeline.  
These outputs demonstrate the intermediate and final results used for analysis, validation, and reporting.

---

## 1. Intermediate Outputs

### 1.1 half_month_data.csv
- Preprocessed dataset at half-month granularity  
- Contains cleaned and structured bug-level information  
- Used as the base input for temporal window creation  

---

### 1.2 half_monthly.csv
- Aggregated dataset grouped by:
  - Month
  - Product
  - Status (New, Fixed, Assigned, Reopened)
- Represents half-month level summaries used for baseline d-set computation  

---

### 1.3 bimonthly_all.csv
- Baseline aggregated dataset (bi-monthly)
- Used to generate **R_star** (reference ranking)
- Acts as a benchmark for comparing temporal window rankings  

---

## 2. d-set Computation Outputs

These files contain normalized values and computed d-set scores for each temporal window.

### 2.1 R1_calc.csv to R6_calc.csv
- Output for temporal windows R1 to R6  
- Each file includes:
  - Product-level normalized attributes  
  - d-set scores  
  - Ranking values  

Interpretation:
- Earlier windows (R1, R2) may show instability  
- Later windows (R5, R6) typically show convergence  

---

### 2.2 R_star_calc.csv
- Baseline ranking computed from bi-monthly data  
- Serves as a reference for evaluating temporal stability  
- Used in correlation analysis  

---

## 3. Ranking Comparison Outputs

### 3.1 rank_compare.csv
- Consolidated ranking table  
- Contains ranks for:
  - R1 to R6  
  - R_star  
- Enables direct comparison across temporal windows  

---

### 3.2 rank_correlations.csv
- Pairwise correlation results between rankings  
- Includes:
  - Spearman correlation (ρ)
  - Kendall correlation (τ)
  - Corresponding p-values  

Interpretation:
- Higher correlation → stronger agreement between rankings  
- Significant p-values (p < 0.05) → statistically reliable relationships  

---

## 4. Key Insights from Sample Outputs

- Rankings stabilize over time (R2 → R6 show increasing agreement with R_star)  
- Early windows (R1) may exhibit noise or variability  
- Correlation analysis validates temporal consistency of rankings  

---

## 5. Usage

These sample outputs can be used for:
- Validating correctness of implementation  
- Demonstrating reproducibility  
- Supporting experimental results in research papers  
- Benchmarking against alternative methods  

---

- Intermediate files should not be modified manually  
- Ensure consistency between input data and generated outputs
